Data Understanding

What data is available or will be collected?
Was there a standard method of collection?
What are the data fields?
Is the data relevant to the problem under consideration?

Learning Outcomes

  1. Recognize the data acquisition and collection process.
  2. Demonstrate data cleaning and manipulation skills.

Data Types

  1. Structured data: highly organized information that can be seamlessly included in a database and readily searched via simple search operations
  2. Unstructured data: it is essentially the opposite of structured data, devoid of any underlying structure

Roughly speaking, a structured data contains labels or fields, which tells us what the number, text or category means. In this course, we will be mostly using structured data.

Data Acquisition

  1. Acquisition: act of acquiring something.
  2. Data Acquisition: acquiring data for analysis.
  3. It could be done via:
    • Direct approach: is a process that converts analog and/or real world data into digital data that can be processed for further analysis. Data Acquisition System (DAS) are used in this acquisition.
    • Indirect approach: is a process of identifying sources/vendors that provides the data based on specifications.

Data Collections

  1. Collections of data available for analysis. For example: Open data repositories and Social media data.
  2. Legality and usage rights should be considered before using data from collections.

Data Storage

Structured text data (the data we consider in this course) can be stored in the following formats:

  1. Comma Separated Values CSV
  2. Tab Separated Values TSV
  3. eXtensible Markup Language XML
  4. Really Simple Syndication RSS
  5. JavaScript Object Notation JSON

We will be mostly working with the CSV or TSV files in this course. However, data in any of the above forms can be read into Python (or any programming language of your choice) using standard procedures. For example, to read:

  • CSV, TSV: pandas.read_csv
  • JSON: pandas.read_json

Data Preprocessing

Real world data is often dirty and needs preprocessing before its desired usage.

What makes data dirty:

  1. Incomplete
  2. Noise
  3. Inconsistency

Data Scrubbing/Cleaning

Following are different ways to clean data:

  1. Handling missing data: Global constant, Ignoring, Imputing or Inference approach.
  2. Filtering noisy data: Identifying outliers and removing from database.
  3. Munging or Wrangling: Involves reformatting data into usable format.

Data Manipulation/Transformation

Some of the typical data manipulation and transformation techniques are:

  1. Reduction:
    • Data Cube Aggregation
    • Dimensionality Reduction
  2. Discretization:
    • Nominal
    • Ordinal
    • Continuous
  3. Conversions
    • One Hot Encoding
    • Feature Construction
  4. Normalization:
    • min-max
    • Z-score
    • Scaling
    The above techniques do not require heavy dependency on the domain experts. In addition to the above, techniques like: Smoothing, Generalization, and Feature construction are used, which require involvement of domain experts.
